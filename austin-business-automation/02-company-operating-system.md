# Company Operating System (COS)

An EOS/Rockefeller-Habits-style operating system for running the AI-automation
consultancy itself — vision, people, data, process, and traction — sized for a
small team (or solo founder) targeting the West Austin / Dripping Springs
market described in `01-target-market-research.md`.

## 1. Vision

**Core focus.** Help independent, owner-operated businesses in West Austin and
Dripping Springs (event venues, wineries/breweries, professional services,
real estate, home services) recover the revenue and hours lost to manual
front-office and back-office work, using AI automation.

**Niche.** Don't compete as a generic "AI consultant." Compete as *the*
automation partner for the Dripping Springs/West Austin hospitality and
professional-services corridor — local presence, done-for-you implementation
(not just advice), and case studies drawn from the same industries prospects
recognize (wedding venues, wineries, law firms).

**3-year picture (fill in with real targets once first clients are signed):**
number of retainer clients, recurring monthly revenue, number of verticals
with a repeatable playbook, headcount.

**1-year plan:** land 3-5 anchor clients (one per priority vertical), convert
each into a documented case study, and build one repeatable "audit → implement
→ retainer" motion that doesn't depend on custom work per deal.

**Marketing strategy:**
- **Target market:** the six verticals in the research doc, chamber-directory
  sourced.
- **3 uniques:** (1) local, in-person audits rather than remote generic
  consulting; (2) implementation included, not just recommendations; (3)
  vertical-specific playbooks (wedding-venue intake automation, winery
  reservation/loyalty automation, law-firm intake automation) instead of
  one-size-fits-all AI advice.
- **Proven process:** the four-stage engagement model below.
- **Guarantee:** define once the first few engagements validate what's
  deliverable (e.g., a defined hours-saved or response-time improvement within
  90 days).

## 2. People

Even solo, define the seats explicitly so you know what to hire or contract
for as the business grows:

| Seat | Responsibility |
|---|---|
| Business development | Chamber relationships, outreach, discovery calls |
| Audit lead | Runs the process audit (`03-process-audit-playbook.md`) |
| Implementation | Builds/configures the automations (`04-automation-strategy-roadmap.md`) |
| Client success | Onboarding, training, retainer check-ins, renewal |

One person can hold multiple seats early on — the point is to know which hat
is on at any given time and to know which seat to fill first when revenue
allows (typically implementation, so the founder can stay in sales/audit).

## 3. Data (weekly scorecard)

Track 8-12 leading and lagging numbers weekly. Suggested starting set:

- Discovery calls booked
- Process audits completed
- Proposals sent
- Proposal close rate (%)
- New retainer clients signed
- Monthly recurring revenue (MRR)
- Client-reported hours saved / week (aggregate across active clients)
- Average days-to-payment for own invoices (dogfood the automation)
- Client NPS / satisfaction score
- Chamber/referral-sourced leads vs. outbound-sourced leads

Review this scorecard in a weekly pulse meeting; anything off-target for two
consecutive weeks becomes an issue on the issues list below.

## 4. Process ("the way we do it")

The core repeatable process is the engagement lifecycle, documented in detail
in the other two playbooks in this folder:

1. **Prospect** — source leads from chamber directories by vertical
   (Doc 01), prioritize wedding venues/wineries first.
2. **Discover & Audit** — run the structured process audit (Doc 03) to
   diagnose shortcomings and quantify opportunity.
3. **Propose** — turn the audit findings into a prioritized, scoped proposal
   (quick wins vs. core systems vs. advanced automation, per Doc 04).
4. **Implement** — build the agreed automations in phases.
5. **Optimize & Retain** — move the client to an ongoing retainer for
   monitoring, iteration, and support; track hours saved and reuse it as a
   case study for the next prospect in the same vertical.

Document each of these five stages as a standard operating procedure (SOP)
as soon as it's been run more than once, so delivery doesn't depend on one
person's memory.

## 5. Traction (rocks & issues)

- Set 3-5 quarterly "rocks" (priorities) tied directly to the scorecard —
  e.g., "close 2 wedding-venue clients," "document the winery reservation
  automation SOP," "get first case study published."
- Run a weekly pulse meeting: scorecard review → rock progress → issues list
  → next actions. Keep it under 60-90 minutes.
- Maintain a running issues list (blockers, ambiguous scope, tooling gaps)
  and solve the top issue each week rather than letting it accumulate.
